# My Journey as a Java Full Stack Developer

Finally! The day has come. After months of learning, building, and breaking things, I'm excited to share my journey into the world of Java Full Stack Development. From writing my first "Hello World" in Java to deploying microservices on AWS - it's been one heck of a ride! ☕🚀

## How It All Started

It all began at **Savitribai Phule Pune University** during my Computer Engineering degree. I wrote my first Java program and honestly? I was confused. But something about Java's "Write Once, Run Anywhere" philosophy stuck with me.

Fast forward through late-night coding sessions, countless Stack Overflow tabs, and coffee-fueled debugging marathons - here we are!

### The Training Journey

**Teks Academy** (June 2025 - February 2026) was where things got real:
- Built full-stack applications using Java, Spring Boot, and React
- Learned Microservices architecture (finally understood what the hype was about!)
- Got my hands dirty with JDBC, SQL, and AWS deployment

## Real-World Experience

### Infoz IT Solutions (Current 💼)
Currently working as a **Java Full Stack Developer Intern**, where I:
- Develop RESTful APIs using Spring Boot
- Build responsive UI components with React
- Integrate backend services with frontend applications
- Actually use Git properly (no more `git push --force`... mostly)

### JPMorgan Chase Virtual Internship
This was a game-changer! I got to:
- Integrate **Apache Kafka** into a Spring Boot microservice
- Work with **Spring Data JPA** and H2 database
- Build REST endpoints that actually returned proper JSON (unlike my first attempts 😅)

## What I've Learned (The Hard Way)

1. **Spring Boot is powerful** but also wants to be your best friend whether you like it or not
2. **Microservices** aren't just "small services" - they come with their own headaches (but worth it!)
3. **Kafka** is amazing until you forget to configure something and wonder why no messages are flowing
4. **Git** - still haven't mastered it, but at least I don't cry anymore when I see merge conflicts
